# LLM Chat Project

Учебный проект для изучения работы с локальными LLM-моделями через Ollama.

## Цели проекта

1. Развернуть локальную LLM и создать чат-интерфейс
2. Изучить влияние параметров генерации (temperature, top-k, top-p)
3. Понять устройство нейросетей и prompt engineering
4. Реализовать AI-агента с памятью и чтением файлов проекта

## Архитектура

Проект построен по слоёной архитектуре. Интерфейсы (консоль и Telegram-бот) принимают ввод пользователя и передают его в AIAgent — ядро системы. Агент содержит память диалога (AgentMemory), читалку файлов проекта (ProjectReader) и обработчик команд. Для общения с моделью используется LLMClient — абстракция над Ollama API. TokenTracker считает токены и логирует каждый запрос.

Поток обработки запроса:

1. Пользователь вводит сообщение в консоли или Telegram
2. Интерфейс передаёт сообщение в AIAgent.process_message()
3. Агент проверяет, является ли сообщение командой по регулярным выражениям
4. Если команда — выполняет действие: чтение файла, смена промпта, очистка памяти
5. Если обычный текст — добавляет сообщение в память и отправляет всю историю модели
6. Модель через Ollama возвращает ответ с учётом контекста
7. Ответ сохраняется в память и выводится пользователю

## Быстрый старт

### Требования

- Python 3.10 или новее
- Poetry для управления зависимостями
- Ollama для запуска моделей локально

### Установка

Склонируйте репозиторий и перейдите в директорию проекта. Установите зависимости командой `poetry install`. Скачайте модель командой `ollama pull qwen2.5:0.5b`.

### Консольный интерфейс

Запустите `python -m llm_chat.console_app` для работы с настройками по умолчанию. Чтобы указать модель и путь к проекту, используйте флаги: `python -m llm_chat.console_app -m qwen2.5:0.5b -p /path/to/project`.

### Telegram-бот

Скопируйте `.env.example` в `.env` и укажите токен бота. Затем запустите `python -m llm_chat.bot`.

## Возможности агента

### Команды

- `/read <путь>` — прочитать файл или директорию. Пример: `/read .`
- `/file <путь>` — прочитать конкретный файл. Пример: `/file src/main.py`
- `/system <текст>` — задать системный промпт. Пример: `/system Ты — senior разработчик`
- `/system` — показать текущий системный промпт
- `/roles` — список доступных ролей
- `/clear` — очистить историю диалога
- `/memory` — статистика памяти агента
- `/stats` — статистика токенов

### Команды на естественном языке

Агент понимает русские формулировки: `прочитай проект .`, `покажи файл bot.py`, `найди ошибку в сервисе`, `предложи улучшения для кода`.

### Роли

Доступны предустановленные роли, которые меняют поведение модели:

- `python_expert` — Senior Python-разработчик с 15-летним опытом
- `code_reviewer` — эксперт по code review, находит баги и проблемы безопасности
- `teacher` — преподаватель, объясняет концепции простыми словами
- `architect` — системный архитектор, проектирует и объясняет архитектурные решения

Для установки роли используйте `/system python_expert`.

### Специальные файлы

При чтении проекта агент автоматически находит и применяет как системный промпт следующие файлы: AGENTS.md, AGENT.md, CLAUDE.md, CURSOR.md, MY-AGENT.md. Если README.md содержит инструкции для AI, он тоже будет применён.

## Эксперименты с параметрами

Проект поддерживает сравнение трёх режимов генерации:

- Точный: temperature 0.2, top-k 10, top-p 0.5. Для фактов и генерации кода
- Сбалансированный: temperature 0.7, top-k 40, top-p 0.9. Для большинства задач
- Креативный: temperature 1.5, top-k 80, top-p 1.0. Для идей и архитектурных решений

## Подсчёт токенов

Используется библиотека tiktoken с кодировкой p50k_base, которая даёт точность около 95% для моделей Qwen. Каждый запрос логируется в JSONL-файлы в директории token_logs. Доступна статистика использования и сравнение стоимости с API: GPT-4, GPT-3.5 Turbo, Claude Opus.

## Структура проекта

- `model.py` — клиент для работы с Ollama API
- `agent.py` — ядро агента: память, команды, чтение файлов
- `bot.py` — Telegram-бот на базе python-telegram-bot
- `console_app.py` — консольный интерфейс
- `token_tracker.py` — подсчёт токенов и логирование

## Конфигурация

Переменные окружения задаются в файле `.env`:

- `TELEGRAM_BOT_TOKEN` — токен Telegram-бота
- `OLLAMA_MODEL` — название модели в Ollama, по умолчанию qwen2.5:0.5b
- `REQUEST_TIMEOUT` — таймаут запроса в секундах, по умолчанию 300

## Дневник разработки

### Что использовали

- Ollama для развёртывания LLM локально
- Qwen 2.5 0.5B как компактную модель для кода и текста
- Streamlit для веб-интерфейса с экспериментами
- python-telegram-bot для интеграции с Telegram
- tiktoken для подсчёта токенов
- Poetry для управления зависимостями

### Что узнали

- Нейросеть — это математическая функция: текст превращается в эмбеддинги, проходит через слои внимания и softmax, затем семплируется в ответ
- Temperature, top-k и top-p — это параметры семплирования, а не самой модели
- Prompt engineering включает техники few-shot, chain-of-thought и role prompting
- Русский текст требует примерно в два раза больше токенов, чем английский
- Локальная модель работает бесплатно, тогда как API-аналоги стоили бы сотни долларов в месяц

### Где AI-инструменты помогли

- Генерация структуры проекта и кода бота
- Объяснение архитектурных решений
- Отладка ошибок с event loop и асинхронностью
- Написание документации

### С чем не справились

- Маленькая модель на 0.5B параметров ошибается в сложных рассуждениях
- Эндпоинт /api/tokenize не поддерживается Qwen в Ollama
- Event loop в Windows потребовал особой обработки
